  • FK UserID

  • Status

  • PK PaymentID
  • FK SubscriptionID
  • Amount
  • PaymentDate

  • Status

  • ArtistName
  • Country
Album
  • PK AlbumID
  • FK ArtistID
  • AlbumName
  • ReleaseYear
Song
  • PK SongID
  • FK AlbumID (nullable)
  • Title
  • Duration
Genre
  • PK GenreID
  • GenreName (UNIQUE)
Song_Genre (Bridge)
  • FK SongID
  • FK GenreID
  • Composite PK
Playlist
  • PK PlaylistID
  • FK UserID
  • PlaylistName
  • CreatedDate
Playlist_Song (Bridge)
  • FK PlaylistID
  • FK SongID
  • AddedOn
ListeningHistory
  • PK HistoryID
  • FK UserID
  • FK SongID
  • PlayedAt

Query Optimization

        
Showing the effect of indexing on ListeningHistory table using EXPLAIN and SHOW PROFILES.

        

            

                
Before Index

                
ALTER TABLE ListeningHistory DROP INDEX idx_history_user;
EXPLAIN SELECT * FROM ListeningHistory WHERE UserID = 3;

                

                    
typekeyrowsExtra

                    
ALLNULL150Using where

                

                
Full table scan. MySQL reads every row.

                
Time: ~0.8ms

            

            

                
After Index

                
CREATE INDEX idx_history_user ON ListeningHistory(UserID);
EXPLAIN SELECT * FROM ListeningHistory WHERE UserID = 3;

                

                    
typekeyrowsExtra

                    
refidx_history_user9NULL

                

                
Index scan. MySQL reads only matching rows.

                
Time: ~0.1ms (8x faster)
